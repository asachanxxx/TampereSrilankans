AI Coding Assistant Spec — Event UI (Responsive, Elegant, UI-Only) — UPDATED
0) Goal

Build a modern responsive event-handling web UI with 3 distinct experiences:

Public (anonymous) browsing

Logged-in user view (same header, user menu + “My Events”)

Admin view (separate layout/menu)

✅ UI only (no real backend, no Supabase integration yet).
✅ Use mock data + strongly typed models.
✅ Elegant, minimal, simple colors, excellent spacing/typography.

1) Tech stack (UI-only MVP)

Next.js (App Router) + TypeScript

Tailwind CSS

shadcn/ui (clean, accessible components)

lucide-react icons

Optional: clsx + tailwind-merge

2) Project setup instructions
Create app
npx create-next-app@latest event-ui --ts --app --eslint --tailwind
cd event-ui

Add shadcn/ui
npx shadcn@latest init


Add components:

npx shadcn@latest add button card badge dialog dropdown-menu tabs avatar separator input textarea skeleton toast sheet table

Icons
npm i lucide-react


Optional helpers:

npm i clsx tailwind-merge

3) Design requirements (UI/UX rules)

Visual style

Minimal, elegant, modern; simple neutral palette

Soft background, white cards, subtle borders

Rounded corners (xl/2xl), consistent spacing

Use one subtle accent (for primary buttons/links)

Responsive

Mobile-first

Header collapses into hamburger on mobile

Event grid: 1 col mobile → 2 col tablet → 3 col desktop

Admin sidebar collapses into a slide-over (Sheet) on mobile

Accessibility

Keyboard-friendly dialogs/menus

Clear focus states (don’t remove outlines)

Micro-interactions

Subtle hover + transition only (no heavy animations)

4) Information architecture (routes)
Public

/ → landing (org section + event list)

/events/[id] → event details (public)

Logged-in user

/me → “My Events”

/me/events/[id] → event details + ticket block

Admin

/admin → admin dashboard (events list)

/admin/events/[id] → admin event overview + tabs
